1. THE STEALTH CALCULATOR GATE (CalcX)
--------------------------------------------------------------------------------
The primary defense layer. InnerOrbit prioritizes "social privacy" by appearing 
as a fully functional calculator (CalcX) on mobile devices.

A. DISGUISE LOGIC:

   - On every "Fresh Launch" (app cold-start), the app defaults to Stealth Mode.
   - The user interface is completely replaced with a calculator/game hybrid.
   - All sensitive notifications and data are hidden until the gate is opened.

B. UNLOCK TRIGGERS (Hardened May 2026):

   - TRIPLE-TAP + HOLD: Tap the "footer" area 3 times and hold on the 3rd tap 
     for 2 seconds to trigger the unlock sequence.
   - MAGIC CODES: Special numeric strings (e.g., "777888+") entered into the 
     calculator trigger specific unlocks or "panic" clears.
   - BIOMETRIC WRAP: Unlocking the gate can be tied to the device's Secure 
     Enclave (FaceID/Fingerprint) for a "Double-Lock" experience.

C. SESSION RESILIENCE:

   - Memory Isolation: The "Unlocked" state is stored in a isolated memory 
     segment. This prevents the app from "snapping back" to the calculator 
     during navigation transitions.
   - Decoupled State: The encryption engine (`encryption-core.ts`) maintains 
     its own session markers to ensure that even if the UI glitches, the 
     cryptographic keys remain locked until explicitly authorized.

2. HIGH-ASSURANCE CRYPTOGRAPHY
--------------------------------------------------------------------------------
InnerOrbit uses a multi-layered, post-quantum cryptographic stack.

A. PROTOCOL V6 (PQXDH):

   - Combines Signal's Double Ratchet with ML-KEM-768 (Kyber) protection.
   - Every message is resistant to future quantum computing attacks.
   - Key rotation happens automatically every time a reply is received.

B. CASCADING FALLBACK (Waterfall):

   - The system automatically handles decryption across platforms using a 
     fallback chain: v6 -> v5.5 -> v5 -> v3.5.
   - Legacy unauthenticated modes (v1-v3) are supported for ARCHIVE READ ONLY.
   - Writing is strictly limited to authenticated, hardened protocols.

C. DETERMINISTIC SIV (v3.5):

   - Our baseline protocol uses AES-256-GCM-SIV.
   - This prevents key leakage even if the device's random number generator 
     is compromised (Misuse-Resistance).

3. THE APP LOCK & BACKGROUND PROTECTION
--------------------------------------------------------------------------------
A. TRANSIENT BACKGROUND BUFFER (2s Grace Period):

   - Toggling the volume or checking notifications does not instantly lock.
   - If the user is gone longer than 2 seconds, the Calculator Gate closes 
     and the encryption sessions are purged from active memory.

B. HARDWARE BINDING (HSM/TPM):

   - Master Keys are never stored as plaintext on disk.
   - On Windows, we use TPM 2.0 / DPAPI.
   - On Mobile, we use the Secure Enclave / Strongbox.
   - If the hardware ID changes, the keys become permanently unreadable.
